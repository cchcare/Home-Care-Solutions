Strict Time Management System (Clock In/Out)

1. Core Rules (Strict Mode)

Implement rules that prevent misuse.

Clock-In Rules
	•	Employee must be scheduled for that shift
	•	Allow clock-in only 10 minutes before shift
	•	Deny if already clocked in
	•	Require GPS location
	•	Require device ID or kiosk mode

Clock-Out Rules
	•	Cannot clock out before minimum shift time
	•	Require break confirmation
	•	Require GPS verification
	•	Manager approval if outside schedule

Other Controls
	•	Prevent editing times without manager approval
	•	Log all actions (audit trail)
	•	Detect overlapping shifts
	•	Lock records after payroll export

3. Strict Validation Logic

Example backend logic:

Clock In

IF current_time < shift_start - 10 minutes
    deny("Too early to clock in")

IF current_time > shift_start + 30 minutes
    mark_late()

IF already_clocked_in
    deny("Already clocked in")

verify GPS distance < 200 feet

Clock Out

IF no clock_in
    deny("You must clock in first")

IF shift not finished
    require manager approval
